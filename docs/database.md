# OpenNum Database Notes

The hosted API expects Supabase/Postgres tables for registrations and public guestbook messages.

## Auth nonce table

Server write endpoints should use one-time nonces before accepting wallet signatures. Nonces expire quickly and are marked as used by the shared auth verifier.

```sql
create table if not exists auth_nonces (
  nonce text primary key,
  wallet text not null,
  action text not null,
  used boolean not null default false,
  created_at timestamptz not null default now(),
  expires_at timestamptz not null
);

create index if not exists auth_nonces_expiry_idx
  on auth_nonces (expires_at);

create index if not exists auth_nonces_wallet_action_idx
  on auth_nonces (wallet, action, used);
```

## Registrations migration

Older deployments only stored `inscription_txid`. Add `inscription_id` so non-`i0` inscriptions can render the correct avatar, content preview, and market link.

```sql
alter table registrations
  add column if not exists inscription_id text;

update registrations
set inscription_id = inscription_txid || 'i0'
where inscription_id is null
  and inscription_txid is not null;

create index if not exists registrations_inscription_id_idx
  on registrations (inscription_id);

alter table registrations
  add column if not exists bio text;

alter table registrations
  add column if not exists for_sale boolean not null default false,
  add column if not exists ask_note text,
  add column if not exists satflow_url text;

create index if not exists registrations_for_sale_idx
  on registrations (for_sale)
  where status = 'active';

-- One wallet can have only one active OpenNum ID.
update registrations
set status = 'unbound',
    updated_at = now()
where status = 'active'
  and wallet_address = 'bc1pyzgyfjaw26cmwdx8px3t92zvhacgx0098mmvgnay4hc2uh0quafsgsp82m'
  and inscription_num in (9727, 64597707);

create unique index if not exists registrations_active_wallet_unique_idx
  on registrations (wallet_address)
  where status = 'active';

create table if not exists inscription_collections (
  id bigint generated by default as identity primary key,
  inscription_id text not null,
  collection_slug text not null,
  collection_name text not null,
  source text not null default 'manual',
  verified_at timestamptz not null default now(),
  created_at timestamptz not null default now(),
  unique (inscription_id, collection_slug, source)
);

create index if not exists inscription_collections_inscription_id_idx
  on inscription_collections (inscription_id);
```

The API keeps a fallback path for older schemas, but the migration should be applied before relying on profile avatars, marketplace links, or owner selling intent.

## Holder periods and profile versions

Social data belongs to a holder period, not just to a number or wallet. These tables create the ownership foundation for messages, follows, offers, and future transfer history.

```sql
create table if not exists holder_periods (
  id bigint generated by default as identity primary key,
  inscription_num bigint not null,
  wallet_address text not null,
  started_at timestamptz not null default now(),
  ended_at timestamptz,
  is_current boolean not null default true,
  start_reason text not null default 'claim',
  end_reason text,
  created_at timestamptz not null default now()
);

create unique index if not exists holder_periods_current_unique
  on holder_periods (inscription_num)
  where is_current = true;

create index if not exists holder_periods_num_idx
  on holder_periods (inscription_num, started_at desc);

create index if not exists holder_periods_wallet_idx
  on holder_periods (wallet_address);

create table if not exists profile_versions (
  id bigint generated by default as identity primary key,
  holder_period_id bigint not null references holder_periods(id),
  inscription_num bigint not null,
  display_name text,
  bio text,
  links jsonb default '{}'::jsonb,
  for_sale boolean not null default false,
  ask_note text,
  satflow_url text,
  is_current boolean not null default true,
  created_at timestamptz not null default now()
);

create index if not exists profile_versions_period_idx
  on profile_versions (holder_period_id, created_at desc);

create unique index if not exists profile_versions_current_unique
  on profile_versions (holder_period_id)
  where is_current = true;

alter table registrations
  add column if not exists current_owner text;

alter table registrations
  add column if not exists owner_checked_at timestamptz;

insert into holder_periods (inscription_num, wallet_address, started_at, start_reason)
select r.inscription_num, r.wallet_address, coalesce(r.registered_at, now()), 'register'
from registrations r
where r.status = 'active'
  and not exists (
    select 1
    from holder_periods hp
    where hp.inscription_num = r.inscription_num
  );

insert into profile_versions (
  holder_period_id,
  inscription_num,
  display_name,
  bio,
  links,
  for_sale,
  ask_note,
  satflow_url
)
select
  hp.id,
  r.inscription_num,
  r.display_name,
  r.bio,
  coalesce(r.links, '{}'::jsonb),
  coalesce(r.for_sale, false),
  r.ask_note,
  r.satflow_url
from registrations r
join holder_periods hp
  on hp.inscription_num = r.inscription_num
 and hp.is_current = true
where r.status = 'active'
  and not exists (
    select 1
    from profile_versions pv
    where pv.holder_period_id = hp.id
      and pv.is_current = true
  );

alter table holder_periods enable row level security;

drop policy if exists holder_periods_anon_read on holder_periods;
create policy holder_periods_anon_read
  on holder_periods
  for select
  to anon
  using (true);

alter table profile_versions enable row level security;

drop policy if exists profile_versions_anon_read on profile_versions;
create policy profile_versions_anon_read
  on profile_versions
  for select
  to anon
  using (true);
```

## Guestbook table

Guestbook messages are public, off-chain, and wallet-signed.

```sql
create table if not exists guestbook (
  id uuid primary key default gen_random_uuid(),
  inscription_num bigint not null,
  parent_id uuid references guestbook(id) on delete cascade,
  message text not null check (char_length(message) <= 280),
  author_wallet text not null,
  author_number bigint,
  signature text not null,
  signed_message text not null,
  created_at timestamptz not null default now()
);

alter table guestbook
  add column if not exists parent_id uuid references guestbook(id) on delete cascade;

create index if not exists guestbook_inscription_num_created_at_idx
  on guestbook (inscription_num, created_at desc);

create index if not exists guestbook_parent_id_idx
  on guestbook (parent_id);

create index if not exists guestbook_author_wallet_idx
  on guestbook (author_wallet);
```

If row-level security is enabled, expose guestbook reads to anon clients and keep writes routed through the serverless API.

## Guestbook holder periods and status

Guestbook messages are tied to the holder period that owned the profile when the message was posted. When a number transfers, old-period messages stay archived with their original holder period instead of following the new holder.

```sql
alter table guestbook
  add column if not exists holder_period_id bigint references holder_periods(id);

alter table guestbook
  add column if not exists status text not null default 'active';

alter table guestbook
  add column if not exists status_changed_at timestamptz;

create index if not exists guestbook_period_status_idx
  on guestbook (holder_period_id, status, created_at desc);

update guestbook g
set holder_period_id = hp.id
from holder_periods hp
where hp.inscription_num = g.inscription_num
  and hp.is_current = true
  and g.holder_period_id is null;
```

`guestbook` already uses RLS from the Phase 0.5 hardening step. These columns inherit the same policy: anon can read public rows, and writes still go through the service-role API.

## Relationship tables

M6 adds the relationship layer for follows, number watches, private watchlists, blocks, and reports. Public relationship tables expose anon reads only; private moderation and watchlist tables have RLS enabled with no anon policies, so reads and writes must go through the service-role API.

```sql
create table if not exists follows (
  id bigint generated by default as identity primary key,
  follower_num bigint not null,
  follower_wallet text not null,
  target_num bigint not null,
  target_period_id bigint references holder_periods(id),
  status text not null default 'active',
  created_at timestamptz not null default now()
);

create unique index if not exists follows_unique
  on follows (follower_num, target_num)
  where status = 'active';

create index if not exists follows_target_idx
  on follows (target_num, status);

create index if not exists follows_follower_idx
  on follows (follower_num, status);

create table if not exists number_watches (
  id bigint generated by default as identity primary key,
  watcher_num bigint not null,
  watcher_wallet text not null,
  target_num bigint not null,
  status text not null default 'active',
  created_at timestamptz not null default now()
);

create unique index if not exists number_watches_unique
  on number_watches (watcher_num, target_num)
  where status = 'active';

create index if not exists number_watches_target_idx
  on number_watches (target_num, status);

create table if not exists watchlist_items (
  id bigint generated by default as identity primary key,
  owner_wallet text not null,
  owner_num bigint,
  target_num bigint not null,
  note text,
  created_at timestamptz not null default now()
);

create unique index if not exists watchlist_unique
  on watchlist_items (owner_wallet, target_num);

create table if not exists blocks (
  id bigint generated by default as identity primary key,
  blocker_num bigint not null,
  blocker_wallet text not null,
  blocked_num bigint not null,
  created_at timestamptz not null default now()
);

create unique index if not exists blocks_unique
  on blocks (blocker_num, blocked_num);

create table if not exists reports (
  id bigint generated by default as identity primary key,
  reporter_num bigint not null,
  reporter_wallet text not null,
  target_type text not null,
  target_id text not null,
  reason text,
  created_at timestamptz not null default now()
);

alter table follows enable row level security;

drop policy if exists follows_anon_read on follows;
create policy follows_anon_read
  on follows
  for select
  to anon
  using (true);

alter table number_watches enable row level security;

drop policy if exists number_watches_anon_read on number_watches;
create policy number_watches_anon_read
  on number_watches
  for select
  to anon
  using (true);

alter table watchlist_items enable row level security;
alter table blocks enable row level security;
alter table reports enable row level security;
```

`follows` tracks identity follows and should be closed when the target holder period transfers. `number_watches` tracks the number itself and survives holder changes. `watchlist_items`, `blocks`, and `reports` are private service-role tables; do not add anon policies for them.

## Private offers

M7 adds private buy-intent offers. Offers are tied to the target number's current holder period, so an offer received by one holder does not become visible to a future holder after transfer. OpenNum stores intent metadata only; funds and execution stay outside OpenNum.

`private_offers` is a private service-role table. Enable RLS and do not add anon policies.

```sql
create table if not exists private_offers (
  id bigint generated by default as identity primary key,
  buyer_num bigint not null,
  buyer_wallet text not null,
  target_num bigint not null,
  target_period_id bigint not null references holder_periods(id),
  price_text text,
  note text,
  satflow_url text,
  status text not null default 'open',
  signature text not null,
  created_at timestamptz not null default now()
);

create index if not exists private_offers_target_period_idx
  on private_offers (target_period_id, status, created_at desc);

create index if not exists private_offers_buyer_idx
  on private_offers (buyer_num, created_at desc);

alter table private_offers enable row level security;
```

`private_offers` intentionally has no anon policy. Buyers and sellers access it only through signed `/api/social` actions using the service-role API.
